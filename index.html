1.1 The Purpose of IT in Modern Business

IT is, by now, a completely integrated way of working in mostly all current businesses. Before the days of IT data was all paper based and records had to be physically transferred from one place to another. IT allows us to digitally transfer data from place to place instantly and without delay, this means that IT can and has exponentially increased the efficiency with which businesses can operate. IT can also increase security in business with the use of encryption and passwords. Completely online, secure forms of payment and paperless business transactions are now possible.[1]

There is however an efficient and non-efficient way to integrate IT into a business. An efficient way is to integrate IT in such a way so either day to day or more involving tasks are quickened or eliminated from a work process by the inclusion of IT. An inefficient way to integrate IT would be to add extra tasks or extra dependencies on certain tasks to the way a business operates, thus increasing the amount of time tasks take and decreasing efficiency over all.[2]

1.2 Methods, Skills and Resources Needed to Complete Digital Projects Successfully

Workflow management applications such as Trello can help to organise tasks and compartmentalise them into smaller, structured chunks as well as scheduling tasks to be completed at certain times or by certain people. Trello is useful for using the Kanban method of project management in which tasks are organised into ‘to do’, ‘doing’ and ‘done’ sections.[3] Slack is also a useful messaging tool for sharing code and files and can also be used for automating features from other apps.[4]

When planning a digital project it is also important to delegate and organise tasks, It is also important to get the right people with the right skills for the job. These skills might be expertise in certain systems, programming languages or generalised areas of expertise. With the right people on the job the task will be completed in a relatively speedy manner. If working independently, it is important to organise tasks in terms of priority and dependencies so tasks can be completed in the right order.

1.4 Describe the risks that might impact digital projects - what causes digital projects to fail? Why?

Deadlines may be strict and not be met in time. This could be for reasons of poor management, poor organisation or because of unforeseen problems that halt the projected speed of the project. This could result in pulling the plug on a project if it is incredibly time critical.

A project that loses its focus can be a problem and can result in not only slow progress but a less than optimal outcome. One of the problems that could occur in such case is feature creep. This is when, either a vagueness in the goals for the project or an over zealous workforce, results in the development of too many features for what is required for the project outcome. This results in the project taking far too much time and having too many, overcomplicated systems.[6] Another problem is the opposite, this is where only the bare minimum is created for the project. This could result in an application which requires too much involvement by the end user and wastes too much time for them.

Security could be an issue. It is important that all software designed is tested for any leaks in security. If this is not done correctly software is at risk from attacks or hackers.[7]

Lack of bug fixes or after support could also be an issue. Even after apps and software are tested and released there still may be bugs and it is important to have a facility to gather error reports and/or listen to user comments. Many apps and software also need updates regularly in order to keep up with current generations of OSs and devices. If these are not kept up with, software will become unsupported.[8]

1.5 Describe how you would go about selecting and using IT systems and software

If it is designing an app for example you have to decide whether it will be native or cross-platform and what kind of programming languages or applications would be best suited to develop that app. The Xcode development environment and Swift programming language could be used if developing for iOS for example.[9] If developing for Android, Android Studio development environment and C++, Kotlin or Java programming languages could be used.[10]

If developing a cross platform app, you need a server to host your web based content. You then have to choose which languages to use to generate your content. You also need to ensure that the way you are working ensures that the app is compatible with multiple systems and have no native features.[11]

If it is a web application for example, you have to take into account what setup you want for your server and what languages you want to use for server/client side processes or whether it can be integrated into a larger framework.[12] You might also want to decide what web hosting platform you might want to use such as Amazon Web Services, Microsoft Azure.[13][14]

1.7 Describe legal guidelines and constraints that impact digital projects

The GDPR or General Data Protection Regulations are very important when designing software and apps. They ensure that sensitive and/or personal information cannot be traded or used unfairly or without consent. This ensures people’s privacy and holds companies liable for any breach of any rules. The new European Data Protection Regulations enforce that any website that uses cookies or stores user data of any kind has to warn and ask for permission before any data from the user is stored.[15]

2.3 Assess the strengths and weaknesses of your final project

Due to a lack of understanding of the particular way CSS works in coordination with HTML there was a lot of trial and error involved in the creation process of my blog. As a result things were not developed as quickly as I would have liked and I did not get to introduce features that I would have liked. Here are a list of strengths and weaknesses of my website taken from the perspective of a first time viewer:-

Strengths

  • Simple layout, easy to understand.
  • Jump-to links to navigate easily to references.
  • Back-to-top links to navigate easily from any of the main headings to title bar.
  • Pleasing blue colour on main title bar.
  • Nice background image of Dovestones Reservoir.
  • Tab bar and all divs slightly translucent so background image visible.
  • Text visible against background through the use of contrasting colours and shadow effects.
  • Titles and subtitles clearly delineated from one another by being different sizes.
  • The whole page and all its content, including the menu and the background, scales to window size and does not distort or come out of place (unless the window is incredibly small).

Weaknesses

  • Style and colours of website are very basic.
  • No rollovers.
  • The one page style appeals to me but may not appeal to all users.
  • Only one font used for whole blog, although this creates a consistent style throughout (and was a conscious decision), the lack of other text styling leaves the page looking a bit bare.
  • When clicking on the references, although the jump-to links work well, the references themselves are not highlighted.
  • When clicking on the number for a reference in the reference section it does not take you back to the position in the text where the reference was.

3.1 Review the benefits and drawbacks of IT tools and systems used in terms of productivity and efficiency

Trello

The benefits of trello are that you can easily format your tasks in terms of priority and both organise and update your workflow very easily. Another benefit is that you can easily work collaboratively by assigning tasks to people and giving different people permission to edit the board. A drawback is that it is always online and if an internet connection is lost then it is not possible to see the updated board or even to schedule updates for a later time. Another drawback is push notifications for phones work incredibly slowly so using your phone as a reminder for trello is not an option.

VS Code

The benefits of VS code are that it automatically detects the language you are using and adjusts its syntax detection accordingly. The autofill feature saves time by recommending objects, functions or variables you have used previously. You get tooltips on objects or variables you use the most useful for me was the colour picker tooltip in CSS. You can also push easily to Github with the inbuilt terminal. The inbuilt debugger allows you to run and debug code without having to go through external means. The drawbacks of VS Code is that certain extensions have to be downloaded in order to access features. This can also be a benefit though as it keeps the application size at a minimum and means that there are no superfluous processes being run.

1.5 Describing copyright and other constraints affecting websites

Copyright law means that work under its protection cannot be copied, plagiarised, distorted or misused by any person or group of people without the original copyright holder’s consent. Work can be paraphrased or referenced but only as long as the original copyright holder is alluded to.[16][17] There is also Fair Use, this means that within reason an original copyrighted work can be used if it is sufficiently adjusted (as in artwork or image), commented upon (for the purpose of review or critique for example) or used for demonstration purposes (teaching, documentary etc) this counts as Fair Use.[18] There is also the concept of Copyleft which is where some of the rights of the property holder are waived deliberately to allow the property to be freely copied, shared and adapted provided that all subsequent iterations of the property are kept under the Copyleft protection.[19] All these things have to be taken into account when publishing a website. For this reason I have ensured that I have referenced all my sources and not plagiarised any content.

1.7 Describe which are the appropriate filetypes for websites

The appropriate file types for websites, and the most commonly used, are, for front end: CSS, HTML and JavaScript.
